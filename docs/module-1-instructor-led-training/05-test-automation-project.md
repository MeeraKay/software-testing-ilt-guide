# 5. Test Automation Project

## Test Automation project

Test Automation projects are used to perform application testing.

!!! warning "Access required"
    Only **App Test Developer** users can create and publish Test Automation projects.

## Overview

- Application Testing in Studio supports both VB and C#.
- Application testing is implemented through test cases and data-driven test cases.
- Test automation projects can contain multiple test cases, providing multiple execution entry points when tests are organized for sequential execution.
- By default, execution occurs at the individual test case level, unless additional XAML files are explicitly invoked.
- Existing workflows can be converted into test cases, imported from other projects, or created from scratch.
- When you publish test automation packages, a corresponding process is automatically created in Orchestrator.

## Included packages

Test Automation projects come preconfigured with the **Testing Activities** package, enabling the creation and execution of automated tests without additional setup.

## Key components

Test Cases, Data-Driven Test Cases, Test Data, and Test Results.
